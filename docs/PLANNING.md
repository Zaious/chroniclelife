# ChronicleLife — 桌面倒數時間軸 Widget 規劃書

> 版本:v0.1(初版規劃)
> 日期:2026-07-10
> 平台:Windows 優先,架構預留跨平台(macOS / Linux,遠期行動裝置)
> 模式:本機離線,無帳號、無雲端

---

## 1. 產品概述

一個釘選懸浮在桌面邊緣的輕量 Widget,以「會隨時間縮短的橫向長條」呈現多個截止日倒數:

```
[═══════════════════════════════] 7/15 論文交稿
                     [══════════] 7/12 購物確認
                            [▓▓] 7/11 繳電費      ← 進入 24h 倒數,顯示 HH:MM:SS,可閃爍
```

- 條的**長度 ∝ 剩餘時間**,每天縮短,直到截止日歸零。
- 越接近截止的項目**排在越下方**(隨著日期往底縮的視覺隱喻)。
- 條**錨定在停靠側**(靠左停靠 → 條齊左往右長;靠右停靠 → 條齊右往左長)。
- 截止日當天(或設定為 48h 內)切換成**即時倒數計時**顯示。
- 分類系統:使用者自訂分類,每個分類一個顏色,條以分類色渲染。

---

## 2. 核心視覺規則(時間軸條的數學)

### 2.1 長度映射

```
remaining = deadline - now          (毫秒)
window    = 設定值「時間視窗」預設 30 天

ratio  = clamp(remaining / window, 0, 1)
bar_px = max(ratio * MAX_BAR_WIDTH, MIN_BAR_WIDTH)
```

- 超過視窗(如 60 天後)的項目 → 滿版長條(或設定為隱藏,見設定項)。
- `MIN_BAR_WIDTH`:即使剩 1 小時也保留一小截,避免看不見。
- v1 用**線性**映射;log 映射(近期項目差異放大)留作設定選項(v1.x)。

### 2.2 排序與定位

- 由上而下:**剩餘時間多 → 少**(最緊急的在最底部)。
- 已過期項目:置底、變灰/變紅(設定),顯示「已逾期 N 天」,需手動完成或刪除。

### 2.3 標籤位置(Label Position)

- **切邊(預設)**:標籤固定對齊停靠側邊緣(所有標籤垂直切齊),長條從標籤旁往另一側延伸。優點:文字永遠在同一位置,好掃視。
- **條尾端**:標籤跟在條的尾端(遠離停靠側的那端),隨條縮短而移動。優點:標籤位置本身就是時間的視覺提示。
- 兩種模式皆為設定項,切換即時生效;錨定方向隨停靠側(左/右)自動翻轉。

### 2.4 檢查點(Checkpoints)

每個任務可設多個檢查點(如「7/13 初稿完成」)。檢查點在條上畫一條**垂直細線**:

```
線的位置(距錨定端) = (deadline - checkpoint_time) / window * MAX_BAR_WIDTH
```

- 條的錨定端代表截止日,線的位置**固定不動**;條的尾端(代表現在)每天縮短、逐漸逼近直線 → **尾端碰到線 = 檢查時間到了**,視覺本身就是倒數提醒。
- 未到:細線(分類色加深);已到未確認:變醒目色+可選閃爍;已確認(打勾):變淡/打勾記號。
- hover 直線顯示檢查點名稱與時間(tooltip)。
- 檢查點時間必須介於「現在 ~ 截止日」之間;超出條長(早於視窗)的檢查點暫不顯示,進入視窗後浮現。

### 2.5 列互動(Row Interaction)

- hover 整列:微亮底色,顯示操作提示。
- **點擊列 → 開啟編輯 popover**(貼齊該列,不另開視窗):改標題/日期/分類、管理檢查點、標記完成、刪除。
- 點擊空白處或 Esc 關閉;編輯即時生效。

### 2.6 倒數模式(Countdown Mode)

- 觸發:`remaining ≤ threshold`(預設 24h,設定可改 48h / 12h / 自訂)。
- 顯示由 `M/D 標題` 變為 `HH:MM:SS 標題`,每秒更新。
- 可選閃爍效果(整條或僅文字,CSS animation,設定可關)。
- 非倒數模式下**每分鐘**重算一次長度即可(省電);倒數模式才啟動秒級 tick。

## 4. 技術選型

### 4.1 比較

| 方案 | 體積/記憶體 | 跨平台 | 懸浮視窗控制 | 評估 |
|------|-----------|--------|-------------|------|
| **Tauri 2**(Rust + Web 前端) | ~10MB / 低 | Win/macOS/Linux + mobile(beta) | 原生支援 always-on-top、透明、skip-taskbar、無邊框 | ✅ **建議** |
| Electron | ~150MB / 高 | 桌面三平台 | 同樣支援 | ❌ 對常駐小 Widget 太重 |
| Flutter Desktop | 中 | 桌面+行動 | 視窗控制靠第三方 plugin | △ 視窗細節掌控較弱 |
| WPF / WinUI 3 | 低 | 僅 Windows | 最強 | ❌ 違背跨平台目標 |
| Avalonia(.NET) | 中低 | 桌面三平台 | 良好 | △ 備選(若偏好 C#) |

### 4.2 建議組合

- **殼**:Tauri 2(Rust)— 視窗管理、tray、autostart、檔案 IO
- **前端**:**Svelte 5 + TypeScript + Vite**(輕、無 VDOM、常駐低耗;若偏好 React 亦可,架構不變)
- **樣式**:純 CSS(CSS variables 做分類色與主題),不引 UI 框架
- **儲存**:`tauri-plugin-store` 或自寫 JSON 原子讀寫;**不用 SQLite**(資料量 < 數百筆,JSON 足夠且利於未來檔案同步)
- **打包**:Tauri bundler → NSIS/MSI

前置需求:Rust toolchain、Node.js、Windows 上需 WebView2(Win11 內建)。

## 5. 架構與資料模型

### 5.1 分層

```
┌────────────────────────────────────┐
│  前端 (Svelte)                      │
│  TimelineBar / TaskEditor /         │
│  CategoryManager / SettingsPanel    │
│  store: tasks, categories, settings │
├────────── Tauri IPC ───────────────┤
│  Rust 殼                            │
│  window 控制(置頂/停靠/拖曳)         │
│  tray、autostart、JSON 持久化        │
└────────────────────────────────────┘
```

倒數/長度計算全部在前端做(純函式,方便未來任何平台重用);Rust 只管殼與 IO。
**核心計算邏輯寫成獨立 TS module(`core/timeline.ts`),零 DOM 依賴** — 這是跨平台的關鍵防腐層。

### 5.2 資料檔(`%APPDATA%/chroniclelife/data.json`)

```jsonc
{
  "schemaVersion": 1,
  "tasks": [
    {
      "id": "uuid",
      "title": "論文交稿",
      "deadline": "2026-07-15T23:59:00+08:00",  // ISO 8601 含時區
      "categoryId": "uuid-or-null",
      "done": false,
      "createdAt": "...",
      "checkpoints": [
        { "id": "uuid", "label": "初稿完成", "at": "2026-07-13T20:00:00+08:00", "acked": false }
      ]
    }
  ],
  "categories": [
    { "id": "uuid", "name": "學業", "color": "#4F8EF7" }
  ]
}
```

設定另存 `settings.json`(讓資料檔未來可直接被同步工具搬運而不帶裝置偏好)。

## 7. Windows 視窗實作要點

- `decorations: false` + `transparent: true` + `skipTaskbar: true` + `alwaysOnTop`(可切換)
- 拖曳:自訂標題握把區呼叫 `startDragging()`
- 停靠:切換靠左/靠右時,以 `currentMonitor()` 工作區座標吸附至邊緣,並翻轉條的錨定方向(CSS `direction` / flex 對齊)
- 位置記憶:`onMoved` debounce 後存 settings
- 省電:視窗被遮蔽/最小化時暫停 tick(`document.visibilityState`)

## 9. 風險與備註

1. **透明視窗 + WebView2**:Windows 上 Tauri 透明窗偶有裝置相容性問題 → 備案:不透明窗 + 圓角深色底,視覺相同。
2. **時區/DST**:一律存 ISO 8601 含時區,計算用 epoch ms,不做字串日期運算。
3. **未來同步**:資料層已隔離成單一 JSON + schemaVersion,屆時可走 Syncthing/雲碟檔案同步,或再演進成 CRDT。
4. **命名**:暫定 ChronicleLife(沿用目錄名),打包 identifier 建議 `life.chronicle.app` 風格,確定後不宜再改。
